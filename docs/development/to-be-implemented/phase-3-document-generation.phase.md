- **Status**: Awaiting Implementation
- **Target version**: `1.4.0`  (bump: MINOR — new feature; jsonb fields added to node config)
- **PRD**: [`../prd/wayfinder.prd.md`](../prd/wayfinder.prd.md)

- **Mockups**: [`../mockups/FlowAgent Chat.html`](<../mockups/FlowAgent Chat.html>) — document card (`.doc-card` component in the message feed, below the milestone pill)

## 1. Problem

Phase 2 ends with a session that can complete a `generate_document` step
but only shows a placeholder pill. Phase 3 makes that step produce a real,
downloadable DOCX file generated by filling the node's uploaded `.docx`
template with AI-derived content from the session's message history.
The output is rendered inline as a document card with a Download button.

Phase 1 left a disabled "Upload a .docx template" affordance in the
`NodeConfigModal`. Phase 3 wires it up.

## 2. Goals

- Flow owners can upload a `.docx` template file to any `generate_document`
  node via the node config modal.
- When such a node completes, the server extracts the template's
  `{{variable_name}}` placeholders, asks the AI to populate them using
  the session transcript and flow context docs, and produces a filled DOCX
  using `docxtemplater`.
- The generated DOCX exactly matches the uploaded template's formatting,
  styles, tables, and headers/footers.
- The chat UI renders a document card in the message feed containing
  filename, AI-generated 2-line summary, and Download button.
- Clicking Download streams the DOCX with the filename pattern
  `[FlowName]-[NodeName]-[SessionId8]-[YYYY-MM-DD].docx`.
- Document cards re-render on session reload from `app_documents`.
- Generation failure surfaces a toast and a "Generate document" retry button
  on the milestone pill; it does not crash the session.

## 3. Non-goals

- No PDF output — DOCX only.
- No persistent object storage (MinIO) — that is Phase 4. Phase 3 stores
  files in `DOCUMENT_STORAGE_PATH` (local directory, env-var configurable,
  defaults to `./data/`). Files survive container restarts if the directory
  is volume-mounted; this is documented but not enforced until Phase 4.
- No bulk re-generation for past sessions.
- No collaborative template editing — templates are uploaded as finished
  `.docx` files.

### Template upload (node config modal on canvas)

Phase 1 left a disabled "Upload a .docx template" affordance in the
`NodeConfigModal`. Phase 3 wires it up:

- File picker (accepts `.docx` only, max 10 MB).
- On select: `POST /api/flows/[id]/nodes/[nodeId]/template` — stores the
  file at `DOCUMENT_STORAGE_PATH/templates/<nodeId>/<isoTimestamp>-<filename>`,
  runs `docxtemplater` dry-run validation, returns `{ path, filename }` on
  success or `{ error }` with a user-friendly message if tags are malformed.
- On success: writes `document_template_path` and `document_template_filename`
  into `app_flow_nodes.config` jsonb; the modal shows the uploaded filename
  with a "Replace" and "Remove" affordance.
- On upload error: the modal shows the `docxtemplater` validation error inline
  so the author can fix the template.

### Server pipeline

1. `RunTurn` (Phase 2) detects that a completing node has
   `output_type='generate_document'` and a `document_template_path` in its
   config.
2. It calls `GenerateDocument` with `(sessionId, nodeId)` after persisting
   the milestone. Generation runs **after** the milestone pill is committed so
   a generation failure never blocks session advance.
3. `GenerateDocument` loads the template bytes from `IDocumentStorage`.
4. Runs `docxtemplater` in dry-run mode to extract the list of
   `{{variable_name}}` tags.
5. Calls `ILanguageModel.generateText` (not stream) with:
   - system prompt = `node.config.ai_instruction` (which must include variable
     descriptions)
   - user content = "Return a JSON object with exactly these keys:
     `[variables...]`. Use this session context: [compact transcript +
     flow context docs]"
6. Model returns structured JSON: `{ [variableName: string]: string }`.
7. `docxtemplater` fills the template with the JSON, producing a DOCX buffer.
8. Buffer written to `IDocumentStorage` under
   `generated/<sessionId>/<filename>`.
9. Calls the AI once more (haiku-class model) for a 2-sentence summary.
10. Inserts a row into `app_documents` with `storage_path`, `filename`,
    `summary`.

If the node has `output_type='generate_document'` but no template uploaded,
generation is skipped and the milestone pill shows a "No template configured"
notice to the session owner — the session continues.

### Download endpoint

`GET /api/documents/[documentId]`:

- Resolves the document; verifies the requester is the session owner, the
  session is shared and the requester is authenticated, or the requester is
  admin.
- Reads bytes from `IDocumentStorage` and streams with
  `Content-Disposition: attachment; filename="<filename>"` and
  `Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document`.
- On missing object: returns `410 Gone` with
  `{ error: 'document_unavailable', hint: 'regenerate' }` and the chat UI
  shows a "Regenerate" button.

### Document card in chat

Rendered inline after the milestone pill:

- Document icon (Lucide `FileText`).
- Filename.
- 2-line summary (`app_documents.summary`).
- Download button (primary).
- Subtle tooltip (Phase 3): "Documents require `DOCUMENT_STORAGE_PATH` to be
  volume-mounted; Phase 4 moves to durable object storage."

### Example templates

Three example `.docx` files committed to `docs/templates/`:

- `rft-template.docx` — Request for Tender skeleton with
  `{{project_title}}`, `{{background}}`, `{{scope}}`, `{{evaluation_criteria}}`,
  `{{conditions}}`, `{{timeframes}}`, etc.
- `evaluation-report-template.docx` — Evaluation Report with
  `{{evaluation_date}}`, `{{panel_members}}`, `{{recommendations}}`, etc.
- `contract-management-plan-template.docx` — CMP with
  `{{contract_title}}`, `{{supplier}}`, `{{kpi_table}}` (loop section), etc.

These are starter templates only. Flow owners upload and adapt them via the
node config modal; they are not auto-loaded into any flow by the build process.

## 6. Database changes

No new tables. The following jsonb fields are added to `app_flow_nodes.config`
(all optional; absent for non-document nodes):

- `document_template_path: string` — storage path to the uploaded `.docx`
- `document_template_filename: string` — original uploaded filename (for display)

Phase 3 writes to `app_documents` (created in Phase 0).

## 7. Acceptance criteria

- [ ] A flow owner opens the node config modal for a `generate_document` node,
      uploads a `.docx` template, and sees the filename displayed in the modal.
      The `app_flow_nodes.config` jsonb contains `document_template_path` after
      save.
- [ ] Uploading a `.docx` with a malformed tag (e.g. `{{unclosed`) returns a
      user-friendly error in the modal and does not save the path.
- [ ] Completing a document-generating step in a test session (any published
      flow with a configured template) produces a row in `app_documents` and a
      file at `DOCUMENT_STORAGE_PATH/generated/...`.
- [ ] The chat feed renders a document card immediately after the milestone
      pill, with the AI-generated 2-line summary.
- [ ] Clicking Download triggers a browser DOCX download with the correct
      filename pattern.
- [ ] The downloaded DOCX opens in LibreOffice / Word and shows the template's
      original formatting (fonts, headers/footers, any tables) with
      `{{variable_name}}` tags replaced by the AI-generated values.
- [ ] Reloading the page re-renders the document card from `app_documents`
      (does not regenerate).
- [ ] Deleting or moving the file at `storage_path`, then clicking Download,
      returns 410 with a "Regenerate" affordance in the UI. Clicking
      Regenerate re-runs the pipeline and the file is downloadable again.
- [ ] A different authenticated user (not session owner, not admin, no shared
      link) requesting the download endpoint gets 403.
- [ ] A node with `output_type='generate_document'` but no template uploaded
      shows a "No template configured" notice on the milestone pill; the
      session continues to the next node without error.
- [ ] Document generation failure (e.g. AI JSON parse error) surfaces a toast
      and does not crash the session. The milestone pill remains with a
      "Regenerate" affordance.
- [ ] `VERSION` and root `package.json#version` = `1.4.0`. `validate.sh`
      passes.

## 9. Risks / open questions

- **DOCX rendering quality** — `docxtemplater` preserves template styles
  but the AI-generated text may not flow well within fixed table cells or
  text boxes. Mitigation: review the first generated document manually after
  Phase 3 completes; adjust variable granularity in templates if needed.
- **Token cost of full-transcript generation** — generating a 5-page RFT
  with full transcript can be 5–10 k input tokens. Cost per generation is
  documented; no streaming used (faster wall-clock).
- **Summary call cost** — the 2-line summary is a separate, cheap call. If
  latency is noticeable, derive the summary from the first paragraph of the
  AI JSON output deterministically.
- **Storage path not volume-mounted** — if `DOCUMENT_STORAGE_PATH` is not
  volume-mounted, generated files survive only until container restart. This
  is the same documented limitation as Phase 1 context docs. Resolution is
  Phase 4 (MinIO).
