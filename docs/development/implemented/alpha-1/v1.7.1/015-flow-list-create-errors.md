# Bug Fix: flow.list and flow.create Runtime Errors + Playwright Test Failures

## Root Cause Diagnosis

### Runtime errors (flow.list / flow.create)

Both tRPC procedures fail with "Failed to list flows." / "Failed to create flow." because every
Drizzle query against `app_flows` references `expert_role` (added in migration 0005). If that
migration has not been applied to the running database, Postgres rejects the query with a
"column does not exist" error, which the repository catch-block wraps in the generic message.

Secondary cause: the drizzle meta directory was missing snapshot files for migrations 0004 and
0005. Running `drizzle-kit generate` would silently produce a 0006 migration that duplicates the
table-creation already done in 0004/0005, corrupting future migration generations.

### Playwright test failures

The create-flow dialog gained a required **Expert role** field (`#flow-expert-role`) in v1.6.3.
The submit button is `disabled` when `expertRole` is empty. Both `flows.spec.ts` and
`flow-lifecycle.spec.ts` filled only `#flow-name`, so the button stayed disabled, the mutation
never fired, the dialog never closed, and the `not.toBeVisible` assertion timed out.

## Fix Plan

1. **Auto-migrate at startup** — update `instrumentation.ts` to call `runMigrations()` for the
   Node.js runtime on every startup (both dev and production). Idempotent; tracked by
   `__drizzle_migrations` table.

2. **Reconstruct drizzle snapshots** — create `0004_snapshot.json` (0003 state + tables from
   migration 0004, without the 0005 additions) and `0005_snapshot.json` (full current schema).
   Remove the spurious `0006_glossy_makkari.sql` and `0006_snapshot.json` generated by
   running `drizzle-kit generate` against the broken snapshot state. Fix the journal trailing
   comma.

3. **Fill expertRole in Playwright tests** — add `#flow-expert-role` fill before clicking
   "Create flow" in both `flows.spec.ts` and `flow-lifecycle.spec.ts`.
